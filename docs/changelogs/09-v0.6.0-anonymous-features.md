# v0.6.0 - Anonymous features and widget overhaul

# Published: 2026-03-14

## Title

Anonymous voting, posting, and commenting

## Content

Users can now interact with feedback without creating an account, and the embeddable widget gets a complete feature upgrade.

Anonymous voting - Users can upvote posts without signing in. Enabled by default per workspace. IP-based rate limiting prevents abuse.

Anonymous posting - Users can submit feedback without an account. Controlled by a per-workspace feature flag (disabled by default).

Anonymous commenting - Users can comment on posts without signing in. Controlled by a per-workspace feature flag (disabled by default).

Workspace permissions page - New admin settings page under Settings > Permissions to toggle all three anonymous features per workspace.

Widget post detail view - Full post detail with voting, comments, threaded replies, status badges, and relative timestamps inside the embedded widget.

Widget SDK events - Host apps can subscribe to widget events (vote, post:created, comment:created, identify) via the SDK. Supports on/off commands and returns unsubscribe functions.

Widget metadata - Host apps can attach key-value metadata to widget sessions. Metadata is stored on posts created through the widget.

Session transfer - Widget sessions transfer to the portal via secure one-time tokens, so users clicking "View full discussion" arrive already authenticated.

Anonymous display names - Anonymous users receive auto-generated friendly display names (e.g. "Blue Falcon") instead of showing raw identifiers.
