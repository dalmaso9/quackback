# v0.2.0 - MCP server and OAuth overhaul

# Published: 2026-02-16

## Title

MCP server expansion and OAuth overhaul

## Content

The MCP server now covers the full product surface, and OAuth gets a complete UX refresh.

8 new MCP tools - update_changelog, delete_changelog, update_comment, delete_comment, react_to_comment, manage_roadmap_post, merge_post, and unmerge_post. Total tools: 7 to 15. AI agents can now complete full workflows end-to-end.

Redesigned OAuth consent page - Shows the client application's name, logo, and domain instead of a generic prompt. Scope labels are human-readable and OIDC plumbing scopes are hidden.

OAuth role revalidation - The MCP handler re-reads the principal's current role from the database on every request, so role changes take effect immediately instead of at token expiry.

Explicit reactions - Replaced the toggleReaction function across the entire codebase with explicit addReaction/removeReaction calls. Both are idempotent, preventing accidental state flips.

Pinned comments replace official responses - The legacy officialResponse columns have been removed. Existing responses are automatically backfilled into pinned comments during migration.

Cursor-based inbox pagination - The admin inbox now uses keyset pagination instead of offset-based pagination, improving performance on large datasets.
