# v0.6.1-v0.6.5 - Canny importer, caching, and cascade delete

# Published: 2026-03-21

## Title

Canny importer, performance caching, and cascade delete

## Content

Import from Canny, faster page loads via caching, and smarter post deletion that cleans up linked issues.

Canny importer - New API-to-API import pipeline with a Canny adapter for migrating feedback, votes, comments, and users from Canny to Featurepool.

Cascade delete for linked issues - When deleting a post that has linked issues in external platforms, the delete dialog now shows each linked issue with a checkbox to archive or close it. Supports Linear, GitHub, Jira, GitLab, ClickUp, Asana, Shortcut, Azure DevOps, Trello, Notion, and Monday. Each integration's settings page has an "On post delete" option to control the default behavior.

Dragonfly caching - Tenant settings, integration event mappings, and active webhooks are now cached with 5-minute TTL and write-through invalidation, eliminating redundant database queries on every page load and event dispatch.

Performance: MCP and admin - MCP search responses trimmed from ~170KB to ~40KB per page by returning excerpts instead of full content. Admin inbox queries reduced from ~1.4MB to ~200KB per 100-post query by excluding unused heavy columns like embeddings.

Bug fixes:

- Fixed OAuth token exchange 500 error for MCP clients like Claude Code
- Fixed widget vote and reaction highlights not appearing after SSO identify
- Fixed dark theme flicker on admin pages by reading theme from a server-side cookie
- Fixed MCP OAuth posts showing "Unknown" author due to missing JWT claims
- Removed misleading static vote count from integration issue descriptions
- Fixed portal post edit view spacing to match the read-only view
